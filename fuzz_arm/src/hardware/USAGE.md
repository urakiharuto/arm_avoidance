# UDPコントローラ (udp_controller) 使用方法

実機ロボット（ToPoArm）に対して、UDP通信でサーボ指令を送るためのCLIツールです。

## 1. ビルド方法

プロジェクトのビルドディレクトリで以下のコマンドを実行します。

```bash
cmake ..
make udp_controller
```

## 2. 実行方法

PCとロボットをLANケーブルで接続し、ロボットのIP（デフォルト: `192.168.4.40`）にPingが通ることを確認してから実行してください。

```bash
./build/udp_controller
```

## 3. 操作方法

プログラムを実行すると、以下のコマンド入力待受状態になります。

| コマンド | 説明 | 例 |
| :--- | :--- | :--- |
| **`a`** | **Auto Mode / Manual Mode の切替**<br>Auto Modeでは自動的にサイン波の指令（デモ動作）を送信し続けます。 | `a` (Enter) |
| **`m`** (入力不要) | **Manual Mode (数値入力)**<br>`ID 値` の形式で入力すると、そのサーボを指定した角度（単位不明、生値）に設定します。 | `0 100` (ID 0番を100にする)<br>`1 -50` (ID 1番を-50にする) |
| **`r`** | **Reset**<br>全てのサーボ指令値を `0` にリセットします。 | `r` (Enter) |
| **`q`** | **Quit**<br>プログラムを終了します。 | `q` (Enter) |

## 4. 通信仕様

- **送信先IP**: `192.168.4.40` (udp_controller_main.cppで変更可能)
- **ポート**: `8888`
- **データ形式**: カンマ区切りの数値文字列（CSV）
    - 例: `0,0,100,0,0,...`

## 5. 注意事項

- Auto Mode中は手動入力コマンドは無視されます（上書きされます）。手動操作したい場合は `a` を押して Manual Mode に戻してください。
- 終了時は `q` を押してください。
